## Tech Stack

- **Backend (development):** Flask with blueprint architecture
- **Static build:** [`build_static.py`](build_static.py) renders Flask routes
  into static HTML for GitHub Pages
- **Frontend:** semantic HTML, [`academic.css`](app/static/css/academic.css),
  minimal vanilla JavaScript ([`academic.js`](app/static/js/academic.js))
- **Deployment:** GitHub Pages (via `.github/workflows/deploy-pages.yml`)

## Local Development

```bash
git clone https://github.com/Venkatchavan/portfolio_vc.git
cd portfolio_vc

python -m venv .venv
source .venv/bin/activate          # macOS / Linux
# .venv\Scripts\activate           # Windows

pip install -r requirements.txt

# Optional: configure environment variables
cp .env.example .env

# Run the Flask app for development
python run.py
```

## Static Build (GitHub Pages)

The site deployed to GitHub Pages is the **static build** in `dist/`,
generated by `build_static.py`:

```bash
python build_static.py
```

This:

1. Renders all Flask routes (`/`, `/archive`, project pages, etc.) to HTML.
2. Rewrites Flask `url_for(...)` references to relative `./*.html` paths.
3. Copies static assets and per-project pages into `dist/`.
4. Writes `.nojekyll` so GitHub Pages serves files verbatim.

The generated `dist/` directory is then deployed by
[`deploy-pages.yml`](.github/workflows/deploy-pages.yml).

## Content Conventions

- Publication statuses are conservative. Manuscripts not externally verified
  are labelled **"Manuscript in preparation"** rather than "Published" or
  "Accepted".
- Research projects live on the homepage; engineering and creative artefacts
  live on the [Archive](app/templates/archive.html).
- Tone: research-first. The site avoids language like *futuristic*,
  *journey*, *cutting-edge*, or *enthusiast*.
