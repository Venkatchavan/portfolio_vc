ObserveML — Production Observability for LLM Apps

Drop-in 3-line SDK for end-to-end LLM observability with real-time dashboard, multi-language SDKs, and Grafana plugin

View on GitHub

Project Overview

ObserveML is the observability layer that every LLM application is missing. Drop in 3 lines of middleware and every prompt, completion, tool call, and RAG retrieval is captured, traced, and surfaced on a React dashboard backed by ClickHouse. It fills the gap in the LLM tooling ecosystem: framework-agnostic, <3 lines to integrate, works with any LLM provider, and answers: "Is my AI working?"

Built across 6 production sprints from architecture to an intelligence layer with root cause narration, 7-day cost forecasting, and a Grafana plugin. Deployed on Fly.io with full CI/CD via GitHub Actions, PyPI + npm + RubyGems + Maven package releases.

Technologies Used

  • Python SDK (PyPI) + JS/TS SDK (npm)
  • Java SDK (Maven/Gradle) + Ruby SDK (RubyGems)
  • FastAPI + Uvicorn backend
  • PostgreSQL (metadata) + ClickHouse (metrics, MergeTree, 90-day TTL)
  • React 18 + Vite + Recharts dashboard
  • Redis caching + Stripe billing
  • Grafana plugin + Docker + Fly.io + GitHub Actions

What ObserveML Measures

  • Token cost per session — know your LLM bill per user
  • p95 / p99 latency — SLA-grade monitoring per prompt version
  • Prompt version drift — semantic similarity between prompt versions
  • RAG retrieval quality — are retrieved chunks relevant to the query?
  • Hallucination signals — confidence scoring + factual consistency
  • Task completion rate — did the user get what they asked for?

Technical Challenges

  • Designing a framework-agnostic SDK that works with OpenAI, Anthropic, Gemini, Cohere, and custom LLMs
  • Achieving <5ms SDK overhead so observability never impacts production latency
  • ClickHouse schema supporting high-cardinality trace queries at scale with TTL and materialized views
  • Hallucination confidence scoring without ground truth labels in production

Sprint Milestones

  • Sprint 0–1: Architecture, ClickHouse schema, Python + JS SDK v0.1
  • Sprint 2: Alert rules, anomaly detection, dashboard v0.2
  • Sprint 3: Multi-model comparison, regression detection, v1.0 production
  • Sprint 4: Sampling, SSE live feed, percentiles, CSV export, token budget
  • Sprint 5: Teams/RBAC, Stripe billing, GDPR deletion, Java SDK, API key rotation
  • Sprint 6: Intelligence layer — root cause narration, 7-day cost forecast, Grafana plugin, Ruby SDK

Results & Impact

ObserveML v2.0.0 delivers production-grade LLM observability at minimal integration cost. The platform covers the full observability spectrum — from raw infrastructure metrics to semantic hallucination detection — in a single SDK. The multi-language support and framework-agnostic design make it applicable across any LLM-powered application stack. The intelligence layer in v2.0.0 moves beyond dashboards to active insight: automatic root cause narration and predictive cost forecasting give engineering teams actionable signals, not just raw metrics.
