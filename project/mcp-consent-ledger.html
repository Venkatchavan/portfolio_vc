mcp-consent-ledger

Reusable consent & authorization-proof library for defensive MCP servers — verify that a caller actually owns or is authorized to test a target before running any audit.

Download ZIP Latest Release View on GitHub

Project Overview

mcp-consent-ledger gives any MCP tool a small, opinionated way to ask: "Is the caller actually allowed to point me at this target?" If you are building defensive MCP servers — vulnerability scanners, web auditors, repo health checkers, log triagers — you do not want to be the weapon. Before any tool starts probing a domain, URL, or repo, it should demand and verify proof of authorization, and remember it.

Consent is explicit, scoped, time-stamped, locally stored, and easy to verify. There is no telemetry. There is no remote service. The library performs network requests only when you explicitly call a verification function. Tokens are SHA-256 hashed before persistence; well-known fetches are HTTPS-only, GET-only, no-redirect, 5s timeout, with an SSRF guard.

Quick Install

  • Add a .npmrc: @venkatchavan:registry=https://npm.pkg.github.com
  • Authenticate with a GitHub PAT (read:packages scope)
  • npm install @venkatchavan/mcp-consent-ledger
  • Or globally: npm install -g @venkatchavan/mcp-consent-ledger → ships mcp-consent CLI
  • From source: git clone → npm install → npm run build

Proof Methods

  • dns-txt — control of DNS via TXT record at _mcp-consent.<domain>
  • well-known-file — control of a web origin via /.well-known/mcp-consent.txt
  • signed-statement — caller-attested authorization recorded as evidence
  • manual — offline / internal approval with approver name & reference

Public API

  • generateConsentToken(), verifyDnsProof(), verifyWellKnownProof()
  • recordVerifiedConsent(), recordStatementConsent(), recordManualConsent()
  • hasValidConsent(), requireConsentOrThrow() — drop one line into your MCP tool handler
  • listConsentRecords(), revokeConsent(), exportConsentLedger()
  • CLI: mcp-consent token | verify-dns | verify-file | statement | list | revoke | export | path

Safety & Privacy

  • Local-only storage at ~/.mcp-consent-ledger/consent.json (mode 0600)
  • Tokens SHA-256 hashed before persistence
  • No telemetry, no background network calls
  • Well-known fetches: HTTPS-only, GET-only, no-redirect, 5s timeout, 16KB cap, SSRF-guarded
  • TypeScript + Node 18+, zero runtime dependencies

Why It Matters

"Defensive" MCP servers are only defensive if they refuse to run on targets the caller doesn't actually control. Hard-coding that check inside every server leads to inconsistent UX and subtle bypasses. mcp-consent-ledger centralises it: one library, one ledger, one requireConsentOrThrow() call. The companion server safe-web-audit-mcp is the worked example — it can swap its custom consent code for this library and immediately gain DNS, well-known, and signed-statement proof methods.
