phish-triage-mcp

A local-first MCP server for safely analyzing suspicious emails, SMS, and URLs — purely static, no link detonation, no telemetry, no external lookups.

Download ZIP Latest Release View on GitHub

Project Overview

phish-triage-mcp is a defensive Model Context Protocol server that helps users safely analyze suspicious emails, messages, and URLs — without clicking links, detonating payloads, or contacting any external service. LLM assistants are increasingly asked questions like "is this email real?" or "what is this SMS trying to do?"; the naive answer involves the model fetching the URL, which is exactly what an attacker wants. This server exposes safe, purely static analysis tools that an MCP-capable client can call instead.

All output URLs are defanged by default (https://example.com → hxxps[://]example[.]com) and PII is redacted (emails, phone numbers, OTPs, card numbers, Aadhaar, PAN, UPI). If the optional @venkatchavan/safe-evidence-redactor package is installed, it is automatically preferred for stronger redaction.

Quick Install

  • From GitHub Packages: npm install -g @venkatchavan/phish-triage-mcp
  • Run the server (stdio): phish-triage-mcp
  • From source: git clone → npm install → npm run build → node dist/index.js
  • Optional enhanced redaction: npm install @venkatchavan/safe-evidence-redactor

MCP Tools Exposed

  • triage_email — static analysis of raw RFC 5322 email; parses headers, checks SPF/DKIM/DMARC, detects mismatches, lookalike domains, urgency language, anchor-vs-href tricks
  • triage_message — for SMS, WhatsApp, social DMs; detects OTP-theft, UPI/payment scams, fake courier, fake KYC/Aadhaar lures
  • explain_headers — plain-English explanation of email headers and SPF/DKIM/DMARC results
  • defang_text — defangs URLs/domains/emails/IPs and extracts structured indicators

Safety Guarantees

  • Never visits suspicious URLs or follows redirects
  • Never submits forms, credentials, or attachments
  • No remote reputation lookups, no external API calls
  • No telemetry; all analysis happens locally
  • Output URLs defanged by default; PII redacted by default

Tech Stack

  • TypeScript + Node.js 18+
  • Vitest test suite
  • MCP stdio transport — works with Claude Desktop, VS Code Copilot, and any MCP-capable client
  • Published to GitHub Packages with automated release workflow
  • Levenshtein-based lookalike domain scoring + curated brand list

Why It Matters

Phishing triage is one of the most common "can you help me with this?" requests aimed at AI assistants — and the riskiest. phish-triage-mcp turns it into a safe primitive: the agent can confidently answer "this looks like a credential-harvesting attempt because the From / Reply-To don't match, the link is a shortened bit.ly, and the message uses urgency phrasing" without ever touching the suspicious URL. Drop it into Claude Desktop or VS Code in two lines of config and your assistant can triage emails the safe way by default.
