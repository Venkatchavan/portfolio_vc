safe-evidence-redactor

Privacy-first redaction library, CLI, and MCP server for security reports, logs, AI outputs, MCP tool outputs, and bug reports — fully offline, zero telemetry.

Download ZIP Latest Release View on GitHub

Project Overview

@venkatchavan/safe-evidence-redactor removes sensitive evidence — PII, secrets, tokens — from text and JSON while preserving enough context to keep the output debuggable. Use it before pasting logs into a chat, attaching reports to a ticket, or returning data from an MCP tool. It walks nested JSON structures, preserves keys and shape, redacts only values, and reports the JSON paths that were touched.

Three modes (minimal, balanced, strict) let you trade off coverage versus signal. Allowlists support specific domains, JSON field names, regex patterns, and entire categories. The package ships an MCP server (safe-redact-mcp) so any MCP-aware client can call the redactor as a tool over stdio.

Quick Install

  • npm login --registry=https://npm.pkg.github.com --scope=@venkatchavan
  • npm install @venkatchavan/safe-evidence-redactor
  • Globally for the CLI: npm install -g @venkatchavan/safe-evidence-redactor
  • CLI binaries: safe-redact (text/JSON) and safe-redact-mcp (MCP server)
  • From source: git clone → npm install → npm run build

Categories Detected

  • Always: AUTH_HEADER, COOKIE, BEARER, JWT, API_KEY (vendor), URL_SECRET
  • Balanced & strict: EMAIL, UPI, AADHAAR, PAN, Indian PHONE, IPv4, IPv6
  • Strict only: international phone, GENERIC_SECRET
  • JSON walker preserves keys, redacts values, reports JSON paths
  • Stats output: { total, byCategory, paths }

MCP Server Tools

  • redact_text — input { input: string, mode?, allow? }; returns redacted string + structured stats
  • redact_json — input { input: any, mode?, allow? }; returns redacted JSON + structured stats
  • Fully offline — no network calls, no telemetry
  • Works with Claude Desktop, MCP Inspector, and any MCP-aware agent framework

CLI Examples

  • safe-redact file.txt
  • safe-redact report.json --json --allow-domain example.com
  • cat logs.txt | safe-redact
  • safe-redact --mode strict --stats < dump.log
  • TypeScript + Node 18+, full type definitions, zero runtime dependencies

Why It Matters

Whenever you paste a log, a stack trace, or an MCP tool output into a chat with an LLM, you risk leaking customer emails, IPs, JWTs, or vendor API keys into someone else's training pipeline. safe-evidence-redactor is the small, opinionated layer that sits between your evidence and your assistant — and because it ships as both a library and an MCP server, the assistant itself can call it: "redact this and then tell me what's wrong with it". The companion server phish-triage-mcp auto-detects this package and uses it for stronger PII redaction when present.
