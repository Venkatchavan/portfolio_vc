safe-web-audit-mcp

A defensive Model Context Protocol server for consent-based website security header audits — non-invasive, local-first, and ready to plug into any MCP client.

Download ZIP Latest Release View on GitHub

Project Overview

safe-web-audit-mcp is a small, defensive Model Context Protocol server that performs non-invasive, consent-based safety checks on a single URL. It is designed for site owners and authorized testers who want a quick read on their own site's security headers, cookie flags, redirect chain, and TLS posture — without scanning, crawling, fuzzing, or any kind of active testing.

It performs a single HEAD request (with GET fallback), validates every redirect hop, runs an SSRF guard against loopback and private IP ranges, and returns a structured report with severity-tagged findings, positive checks, and an ethical notice. All consent and history data stays local at ~/.safe-web-audit/.

Quick Install

  • npx -y safe-web-audit — recommended for local MCP clients
  • npm install -g safe-web-audit — global CLI
  • docker run --rm -p 8787:8787 ghcr.io/Venkatchavan/safe-web-audit-mcp:latest
  • VS Code: Command Palette → Chat: Install Plugin From Source → paste the repo URL
  • From source: git clone → npm install → npm run build → npm start

MCP Tools Exposed

  • audit_url — non-invasive defensive audit with compliance mapping (OWASP / CIS / ISO / privacy)
  • verify_authorization — DNS TXT, well-known file, or signed-statement consent proof
  • generate_fixes — ready-to-paste config snippets for Nginx, Apache, Caddy, Cloudflare, Vercel, Netlify, Express, Django, Rails, WordPress
  • audit_history — list and diff locally-saved audits
  • generate_template — security.txt, responsible-disclosure page, password policy, backup checklist
  • emergency_checklist — calm, defensive incident-response checklist

Defensive by Construction

  • No crawling, link-following, fuzzing, brute-forcing, login, or form submission
  • No POST / PUT / DELETE / PATCH requests
  • SSRF-guarded — blocks loopback, RFC1918, link-local, IPv6 ULA, and cloud metadata IPs
  • Safe Evidence Mode redacts cookies, tokens, emails, IPs by default
  • Refuses to run unless confirm_authorized: true

Tech Stack

  • TypeScript + Node.js 18+ (20+ recommended)
  • MCP stdio & Streamable HTTP transports (port 8787)
  • Multi-arch Docker images on GHCR (amd64 + arm64)
  • GitHub Actions CI on Node 18 / 20 / 22 + automated release pipeline
  • Plugin manifest (plugin.json + .mcp.json) for one-click VS Code Agent install

Why It Matters

Most "security scanners" you can give an LLM agent are too aggressive to run on production. safe-web-audit-mcp fills the gap: it gives any MCP-aware client (Claude Desktop, Claude Code, VS Code, Cursor, Continue, Cline, Windsurf, Zed) a tool that's safe to call by default. The auth-proof flow ensures the agent can only audit targets the caller actually controls, and the fix generator turns findings into copy-paste config — so the same agent that found the missing Strict-Transport-Security can hand you the exact Nginx snippet to fix it.
